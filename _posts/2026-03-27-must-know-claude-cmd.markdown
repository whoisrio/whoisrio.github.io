# 你必须会使用的 Claude 命令

## 📋 规划与设计

### `/plan`
特性开发前，使用 `/plan` 做好计划

**示例：**
```bash
# 开发新功能前先规划
/plan
# 然后说：我需要给用户管理系统添加一个批量导入功能
```

---

## 💻 开发与代码质量

### `/simplify`
对刚改完的代码做三维检查（代码复用、质量、效率），发现问题直接修掉。特别适合改完一段逻辑后立刻跑一遍，代替手动 review。

**示例：**
```bash
# 改完代码后立即检查
/simplify
# 自动分析：有没有重复代码？逻辑是否最优？性能如何？
```

### `claude --worktree`
并行开发时创建独立的工作树

**示例：**
```bash
# 在不同分支同时工作，互不干扰
claude --worktree
```

---

## 📚 知识管理

### `/insight`
及时更新 `Claude.md`，保持准确而简洁

**示例：**
```bash
# 项目有新约定或架构变更时
/insight
# 然后记录：新增了 Redis 缓存层，统一键命名规范
```

---

## 🧠 上下文管理

### `/context`
观察上下文情况，及时 `compact`

**示例：**
```bash
# 对话较长时检查 token 使用情况
/context
# 发现占用过多时
compact  # 压缩上下文
```

### `/clear`
开启与前文无关的任务时，记得执行 `/clear` 重置 context 窗口

**示例：**
```bash
# 从前端开发切换到完全无关的后端任务
/clear
# 然后开始新任务
```

---

## 💬 快速询问

### `/btw`
在不打断主任务的前提下快速问一个侧问题，适合"两个命令有什么区别"这类单轮旁路问答，不适合需要读仓库或调用工具的问题。

**示例：**
```bash
# 开发中突然想起一个概念问题
/btw map 和 filter 的区别是什么？
# 得到答案后继续主任务
```

---

## 💡 使用建议

1. **新功能开发**：`/plan` → 开发 → `/simplify` → `/insight`
2. **切换任务**：先 `/clear`，再开始新工作
3. **长对话中**：定期 `/context`，必要时 `compact`
4. **并行工作**：使用 `claude --worktree` 创建独立环境
